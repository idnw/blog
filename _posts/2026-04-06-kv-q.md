# Linux内核驱动 / KVM-QEMU虚拟化 高级工程师面试题

> 适用岗位：高级Linux内核工程师 / 高级虚拟化工程师
> 面试时长建议：2.5 ~ 3小时
> 难度：高级（要求有实际内核开发或虚拟化项目经验）

## 第一部分：Linux内核基础与内存管理（共8题）

### 1.1 进程与调度

**Q1.** 请详细描述Linux CFS（Completely Fair Scheduler）的核心数据结构和调度决策过程。vruntime是如何计算的？当一个进程从sleep状态唤醒时，CFS如何处理它的vruntime以避免它独占CPU？

**Q2.** 请解释Linux内核中的RCU（Read-Copy-Update）机制的工作原理。它解决了什么问题？grace period是如何确定的？请给出一个适合使用RCU而不是rwlock的具体场景。

**Q3.** `task_struct`中的`mm_struct`和`active_mm`分别代表什么？内核线程的这两个字段有什么特殊之处？在context switch时如何处理？

### 1.2 内存管理

**Q4.** 请描述Linux内核伙伴系统（Buddy System）和SLAB/SLUB分配器的关系与各自的职责。当调用`kmalloc(64, GFP_KERNEL)`时，内核内部经历了哪些步骤？如果在中断上下文中应该使用什么标志？为什么？

**Q5.** 请解释Linux的反向映射（Reverse Mapping / RMAP）机制。它在页面回收（page reclaim）中扮演什么角色？匿名页和文件页的反向映射实现有何不同？

**Q6.** 什么是Memory Cgroup（memcg）？请描述当一个cgroup的内存使用达到limit时，内核的处理流程。OOM Killer的选择算法是怎样的？如何在生产环境中调优？

### 1.3 同步原语

**Q7.** 请比较以下内核同步原语的适用场景，并说明各自的实现原理：
- spinlock
- mutex
- semaphore
- rwlock
- seqlock
- per-cpu变量

在什么情况下spinlock会导致死锁？如何检测？

**Q8.** 请解释什么是`preempt_disable()`/`preempt_enable()`，它们与spinlock的关系是什么？在RT（PREEMPT_RT）内核中，spinlock的行为有何变化？

## 第二部分：Linux设备驱动开发（共8题）

### 2.1 驱动模型

**Q9.** 请描述Linux设备驱动模型中bus、device、driver三者的关系和匹配过程。以PCI设备为例，从设备插入到驱动probe被调用，中间经历了哪些步骤？

**Q10.** 请解释Platform Device和Platform Driver的设计目的。在Device Tree场景下，一个platform device是如何被创建的？`of_match_table`和`compatible`属性的匹配规则是什么？

**Q11.** 请详细描述Linux DMA子系统。什么是Streaming DMA和Coherent DMA？`dma_map_single()`和`dma_alloc_coherent()`分别在什么场景下使用？IOMMU/SMMU在其中扮演什么角色？

### 2.2 字符设备与中断

**Q12.** 请实现一个支持以下功能的字符设备驱动框架（伪代码即可）：
- 支持多个设备实例
- 支持阻塞和非阻塞读
- 支持poll/select
- 支持ioctl
- 正确的并发保护

请解释你的设计中每个关键决策。

**Q13.** 请描述Linux中断处理的上半部（top half）和下半部（bottom half）机制。softirq、tasklet和workqueue各自的特点和适用场景是什么？threaded IRQ是什么？什么时候应该使用它？

**Q14.** 请解释Linux的GPIO子系统架构。gpiolib、gpio_chip、gpio_desc之间的关系是什么？pinctrl子系统如何与GPIO子系统协同工作？

### 2.3 调试与性能

**Q15.** 你在生产环境中遇到一个内核soft lockup告警，请描述你的排查思路和方法。会使用哪些工具？如何区分是驱动Bug还是系统负载问题？

**Q16.** 请描述你使用过的内核调试工具和方法，包括但不限于：
- ftrace / function_graph
- kprobe / kretprobe
- perf
- crash工具分析kdump
- eBPF/bpftrace

请各举一个你实际使用过的场景。

## 第三部分：KVM虚拟化（共7题）

### 3.1 KVM架构

**Q17.** 请画出（或描述）KVM的整体架构，说明KVM内核模块、QEMU用户态进程和Guest OS三者之间的交互关系。一次Guest中的I/O操作（如写磁盘）是如何从Guest传递到Host并完成的？

**Q18.** 请详细描述VM Entry和VM Exit的过程。VMCS（Intel）/ VMCB（AMD）中存储了哪些关键信息？列举5种以上常见的VM Exit原因，并说明KVM是如何处理每种Exit的。

**Q19.** 请解释EPT（Extended Page Table）/ NPT（Nested Page Table）的工作原理。与Shadow Page Table相比，硬件辅助页表有什么优势？EPT Violation和EPT Misconfiguration分别在什么情况下发生？

**Q20.** 请描述KVM中虚拟中断的注入机制。以下概念之间的关系是什么？
- Virtual APIC
- Posted Interrupt
- APICv / AVIC
- irqfd / ioeventfd

一个virtio设备的中断是如何从Host通知到Guest的？

### 3.2 KVM高级特性

**Q21.** 请解释KVM的内存管理机制，包括：
- KVM memory slot的概念
- GPA到HPA的转换过程
- KSM（Kernel Same-page Merging）在虚拟化中的应用
- 内存气球（Balloon）驱动的工作原理
- Huge Pages在虚拟化中的性能影响

**Q22.** 请描述vCPU的调度问题。什么是"双重调度"问题（Double Scheduling）？Lock Holder Preemption问题是什么？PLE（Pause Loop Exiting）如何缓解这个问题？在生产环境中如何优化vCPU调度？

**Q23.** 请解释KVM Live Migration的完整过程。Pre-copy和Post-copy迁移各自的优缺点是什么？脏页跟踪是如何实现的？什么情况下迁移可能无法收敛？如何解决？

**Q25.** 请详细解释QEMU中virtio设备的实现。以virtio-net为例，从Guest发送一个网络包到Host物理网卡的完整数据路径是什么？vhost-net是如何优化这条路径的？vhost-user又解决了什么问题？

## 第五部分：综合与实战（共6题）

### 5.1 性能优化

**Q29.** 你负责优化一个KVM虚拟化平台，Guest中运行数据库负载，发现性能只有裸机的60%。请描述你的性能分析方法和可能的优化方案。至少覆盖CPU、内存、I/O、网络四个维度。

**Q30.** 请解释以下虚拟化性能优化技术的原理和适用场景：
- CPU Pinning / NUMA Affinity
- Huge Pages / THP
- virtio-blk vs virtio-scsi vs vfio-pci（NVMe直通）
- SR-IOV
- DPDK / AF_XDP在虚拟化网络中的应用

### 5.2 故障排查

**Q31.** 一台KVM虚拟机频繁出现短暂卡顿（stall），每次持续几百毫秒。Guest内部看不到明显的CPU或I/O瓶颈。请描述你的排查思路。

**Q32.** 一个运行中的Guest突然无法被SSH连接，但QEMU进程仍在运行。请列出可能的原因并描述排查步骤。

### 5.3 架构设计

**Q33.** 请设计一个支持以下特性的轻量级虚拟化方案（类似Firecracker），说明你的技术选型和架构决策：
- 启动时间 < 200ms
- 内存开销 < 10MB
- 安全隔离（多租户场景）
- 支持热迁移

**Q34.** 请比较KVM、Xen、Container（namespace+cgroup）三种隔离方案在安全性、性能、资源开销方面的差异。在什么场景下你会选择哪种方案？请给出具体理由。
